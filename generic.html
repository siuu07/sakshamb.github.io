FANTASY SOCCER GAME

This was probably the best experience of my programming career thus far where I developed a single-player game using Java Swing for GUI that allowed users to add 11 players into their soccer team and earn points through real-world player performance. IDE utilized was IntelliJ IDEA CE.

Once the player joined the game the menu as shown above was displayed. Depending on the option selected the following screens appeared.

Display Team

How it Worked?
Used a "chk" flag to determine whether a team existed before using a getter method to get a list of the Player objects in the Team object. Finally, iterated through the resulting list and displayed each player's name, jersey number and goals.

Add Player

How it Worked?
Checked if team object had a size greater than or equal to 11. If not, then the player object was created using the name, jersey number and goals specified by user. Otherwise, an error message was shown.

Display a specific player's details

How it Worked?
Team list object was parsed through and if the entered name matched an existing player name then their name, jersey number and goals were displayed.

Save/Load Player Data

How it Worked?
Here, I made use of a JsonWriter and JsonReader class that wrote to and read from a local file of type json. Writing was done by calling the following methods - jsonWriter.open(),jsonWriter.write(ob),jsonWriter.close(). On the other hand, reading was done by calling the jsonReader.read() method on the json file.

Update Player Goals

How it Worked?
Team list object was parsed through and if the entered name and the jersey number matched with an existing player then the goals were updated with new number specified by user.

Remove Player

How it Worked?
Team list object was parsed through and if the entered jersey number matched with an existing player then the Player object was deleted from team. Following this, the user was prompted to save changes to Team object.

RESULTS

It wasn't until I had designed the entire game till I realised how much work I had put into this application and how much I had learned through the entire journey. One essential aspect of this project was that of performing testing for each possible scenario one could encounter. I applied my knowledge on exceptions to make sure that the application was robust. Prior to attempting this project I used to develop by getting straight into hard coding and figuring out the relationships between classes as I go along. However, now I realize the efficiency offered by design tools such as class-relationship diagrams, object diagrams and control flow diagrams. This allows me to get an idea of how I can write better code to allow for minimal changes if added functionality is required. After having designed and implemented the main foundation for the game came user-interface design using SWING library. I found this part to be the most interesting as what I had imagined my game to be in the initial stages of the project was gradually coming to life. Using panels, jframes, buttons and textboxes I tried to make the interface as user-friendly as possible.
